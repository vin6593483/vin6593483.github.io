# 📝 小考 (03/12) - 物件封裝與狀態管理：倉鼠 Bibo 體重控制

## 💡 實作重點與解題思路

這次的測驗是標準的**物件導向程式設計 (OOP)** 基礎練習。題目要求建立一個「倉鼠 (hamster)」物件，並透過公開的行為來改變私有的屬性，這正是**封裝** 的核心精神。

### 1. 類別與物件
* **類別 (Class)**：定義了 `hamster` 作為設計圖，規定了倉鼠會有的屬性與行為。
* **物件 (Object)**：主程式中的 `hamster bibo;` 就是根據這張設計圖實際產生出來的倉鼠實體。

### 2. 存取修飾子與資料保護
* **私有屬性 (`private`)**：將 `weight` (體重) 設為私有，代表外部程式碼不能直接修改它（例如無法執行 `bibo.weight = 100;`），藉此保護內部狀態不被隨意竄改。
* **公開方法 (`public`)**：提供了 `set_weight()`、`exercise()` 與 `eat()` 作為對外開放的合法操作接口。

### 3. 狀態管理邏輯
這是一個簡單的狀態機模型，透過特定的函式來改變內部狀態：
* **初始設定**：透過 `set_weight(w)` 寫入起始體重。
* **狀態改變 (進食)**：`eat(f)` 接收食物克數 `f`，使體重增加 `weight = weight + f`。
* **狀態改變 (運動)**：`exercise(e)` 接收運動消耗克數 `e`，使體重減少 `weight = weight - e`。
* **狀態輸出**：`show()` 負責將最終計算好的體重格式化輸出。

---

## 💻 程式碼架構分析

```cpp
class hamster {
private:
	int weight; // 私有屬性：保護體重資料不被外部直接修改

public:
	// 寫入初始體重
	void set_weight(int w) {
		weight = w;
	}
	
	// 行為：運動使其變瘦
	void exercise(int e) {
		weight = weight - e;
	}
	
	// 行為：進食使其變胖
	void eat(int f) {
		weight = weight + f;
	}
	
	// 顯示當前狀態
	void show(void) {
		cout << "Bibo現在" << weight << "公克" << endl;
	}
};
