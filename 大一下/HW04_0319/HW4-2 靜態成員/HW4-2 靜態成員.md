# 🎓 HW4-2：靜態成員 (Static Members) - 會員統計與會費計算系統

## 📝 實作重點與解題思路

本次實作的核心在於練習 **C++ 的靜態資料成員與靜態成員函式**，用來處理不屬於單一物件、而是屬於整個類別的共享資料。

### 1. 靜態成員變數 (Static Data Members)
* **共享特性**：宣告了 `static int count` 與 `static int vip_count`。
* **類別外初始化**：靜態成員必須在類別定義之外進行初始化（如 `int member::count = 0;`），否則編譯時會出現連結錯誤。
* **計數邏輯**：每當建構子被呼叫一次，`count` 就會累加；若傳入的等級為 'v'，則 `vip_count` 也隨之增加。

### 2. 靜態成員函式 (Static Member Functions)
* **無須實例呼叫**：主程式使用了 `member::show_fee()` 的呼叫方式。
* **實作限制**：由於靜態函式不屬於任何特定物件，它內部只能存取「靜態」成員變數（如本例中的兩個計數器），無法存取非靜態的 `id` 或 `level`。

### 3. 原創思路紀錄
* **初始化位置**：特別註記了靜態成員必須在類型外初始化，這是一個非常關鍵的語法細節。
* **主程式陷阱**：識別了題目主程式使用 `member::show_fee()` 呼叫的意圖，並以此推導出必須使用 `static` 宣告。

## 💻 完整程式碼實作與註解牆

```cpp
#include<iostream>
using namespace std;

class member {
private:
	char level;                                             // 私有屬性：會員等級 (v 為 VIP)
	int id;                                                 // 私有屬性：會員編號

public:
	static int count;                                       // 靜態屬性：全體會員總人數計數器
	static int vip_count;                                   // 靜態屬性：VIP 會員人數計數器

	member() {                                              // 預設建構子
    // 空著是因為真的沒啥該建構的，要初始化的count因為是靜態成員所以得在類型外初始化(但要在類型內宣告)
	}

	member(int a, char b) {                                 // 參數建構子：新增會員並分類計數
		id = a;                                             // 設定會員編號
		level = b;                                          // 設定會員等級
		count++;                                            // 每建立一個物件，總人數加 1
		if (b == 'v') {                                     // 判定是否為 VIP 等級
			vip_count++;                                    // VIP 人數加 1
		}
	}

	void show_inf() {                                       // 一般成員函式：顯示人數統計
		cout << "總會員人數:" << count << endl;
		cout << "VIP會員人數:" << vip_count << endl;
	}

	static void show_fee() {                                // 靜態成員函式：計算並顯示總會費
		int sum = (count - vip_count) * 300 + (vip_count * 500);
		cout << "總共會費:" << sum << "$" << endl;
	}
};

// --- 靜態成員變數必須在類別外部進行初始化 ---
int member::count = 0;                                      // 初始化總人數為 0
int member::vip_count = 0;                                  // 初始化 VIP 人數為 0

int main() {
	int a[5];                                               // 儲存輸入的 5 個 ID
	char b[5];                                              // 儲存輸入的 5 個等級
	for (int i = 0; i < 5; i++) {
		cin >> a[i] >> b[i];                                // 批次讀取資料
	}

	// 實體化 5 個會員物件，觸發計數器累加
	member p1(a[0], b[0]), p2(a[1], b[1]), p3(a[2], b[2]), p4(a[3], b[3]), p5(a[4], b[4]);

	p1.show_inf();                                          // 透過物件呼叫一般函式

	// 題目主程式小陷阱:主程式使用了 member::show_fee() 這種"類別名稱::函數"的呼叫方式，
	// 不得不將函數宣告為 static，否則編譯器會因為找不到實體物件而報錯
	member::show_fee();                                     // 透過類別名稱直接呼叫靜態函式

	return 0;                                               // 程式結束
}
