# 作業 2-2：物件行為練習 — 開槍與裝彈模擬 (State Management)

本練習透過模擬槍枝的行為，進一步強化對「類別 (Class)」與「物件狀態」的理解。程式碼實作了子彈數量的封裝，並根據物件目前的狀態（剩餘子彈數）決定不同的輸出行為。

## 🔍 核心觀念解析

### 1. 物件狀態的改變
* **私有屬性 `bullet`**：作為槍枝的狀態，預設為 0 顆子彈。外部無法直接修改子彈數，必須透過類別提供的行為（函數）來改變。
* **行為對狀態的影響**：呼叫 `shoot()` 會使狀態減一（子彈消耗）；呼叫 `reload()` 則會重置狀態（子彈補滿至 3 顆）。

### 2. 條件邏輯與防呆
* 在 `shoot()` 函數中加入了判斷式：若 `bullet == 0`，則不執行射擊動作並提示 `Please Reload`。這模擬了真實系統中「資源枯竭」時的錯誤處理機制。

---

## 💡 完整程式碼

```cpp
#include<iostream>
using namespace std;
class gun {                                           // 定義物件類別
private:                                              // 私有成員，封裝內部資料
	int bullet = 0;                                   // 初始子彈數為 0
public:                                               // 公開成員區
	void shoot() {                                    // 開槍行為函數
		if (bullet == 0) {                            // 判斷是否還有子彈
			cout << "Please Reload" << endl;          // 若無子彈，輸出提示裝彈
		}
		else {                                        // 若有子彈
			bullet--;                                 // 子彈數減 1
            cout << "Bang" << endl;                   // 輸出提示開槍
		}
	}
	void reload() {                                   // 裝彈行為
		cout << "Reload" << endl;                     // 輸出裝彈訊息
		bullet = 3;                                   // 將子彈補滿至 3 顆
	}
};

int main(){                                           
	char a[5];                                        // 建立大小為 5 的字元陣列儲存指令
	gun g1;                                           // 實例化一個物件 g1
	cin >> a[0] >> a[1] >> a[2] >> a[3] >> a[4];      // 讀取五次操作指令 ('s' 為開槍, 'r' 為裝彈)
	for (int i = 0; i < 5; i++) {                     // 依序處理五次操作
		if (a[i] == 's')                              // 若輸入指令為 's'
			g1.shoot();                               // 執行開槍函數
		else if (a[i] == 'r')                         // 若輸入指令為 'r'
			g1.reload();                              // 執行裝彈函數
	}
	return 0;                                         
}
